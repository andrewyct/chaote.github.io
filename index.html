BaybioManager

Developed by:  Chao-te Yang
Written in:   Java

BaybioManager is a desktop application designed for Parisgoshop.com, a cosmetics/organic products vendor.   This back-end management application provides system admin the ability to manage transaction records and monitor inventory status in one place, without having to login to different online stores and manually check the quantities in stock. It also provide a simplfied workflow for the admin's daily operation, reducing the time required to manually insert transaction data into their OpenERP system.  Currently the application is still under development.

From the technical point of view, the implementation of BaybioManager takes the future growth of the company into consideration and it is designed to be easy-to-maintain and easy-to-extend:

  • The GUI is built with a self-implemented MVC-like pattern, which separates the codes for manipulating data and displaying results. It requires extra classes to construct such design pattern, but will be easier to add new components in the future.

  • Data access objects abstracts and isolates the underlying details of database connections, and it will be easier for the developer to switch between different databases or to add new databse connection.

  • High level concurrency api like SwingWorker also makes it simple to separate the EDT and worker threads.




MySurvey

Developed by (sort alphabetically):  Beth Haines,  Chao-te Yang,  Chi-chih Hsu,    Fliden Demiri,  Fyodor Wolf,  Gloria Jensen,  Imaad Khan,  Samuel Tan

Written in:   PHP,  JavaScript
GitHub:   github.com/fydo23/mySurvey
Project site:   ec2-54-191-61-5.us-west-2.compute.amazonaws.com

MySurvey is a semester long team project for the CS673 Software Engineering.  The goal is to create a web application that will provide user the ability to create, take and analyze online surveys.  Our target survey creators will be data analysts from corporations and from the education sector that are collecting data from their designated group of survey-takers.  The application will also provide pre-fabricated tools to speed up survey creation and publication.

There are many related work on the market, some are free , like SurveyMonkey which allow the user to login, create the survey, and view analytic reports, and some are costly but more sophisticated like Visimojo.

MySurvey offers a range of features geared towards both new survey creators and advanced users. This includes question templates to quickly get started in addition to customizable complex branching, time measurement for each question, and fine-tuned analytic reporting. Survey Monkey and Visimojo each offer many of the features we’re planning in our application, but not all.

This project follows a non-strict Agile development process, focused on features and collaboration. There are three iterations (excluding the initial planning phase), three weeks each. The initial phase focuses on planning, organization and defining requirements. The other three iterations are more typical Agile iterations, each of them include the following cycle:
requirement analysis --> implementation and testing --> update and documentation.




MapReduce Focus Area Detector

Developed by:  Chao-te Yang
Written in:   Java

This program detects and highlights the well-focused area of the input images using Hadoop MapReduce framework.  It is a cloud-based program capable of distinguishing the difference between a sharp line and a blurred line, and it can be easily configured and deployed to a computer cluster or the cloud.

If an object is well-focused in the picture, the details on that object should be very sharp and clear because the pixel's RGB value in that area changes rapidly.  If the object is out of focus, the details on that object will be blurred because the pixel's RGB value changes gradually from black to dark gray, from dark gray to light gray, and from light gray to white.

The main task of this program involves calculating the difference of the RGB value between any two consecutive pixels, it is CPU intensive, but the computation can be broken into multiple small tasks and can be processed more efficiently using distributed system.

There are three stages during the MapReduce process.  First, the Builder class writes the pixel information to a text-based file.  Secondly, the Mapper class reads the text-based input, calculates the difference, and writes the output to the next stage.  Finally, the Reducer class calculates the total score and a new image with hightlighted area will be created after this stage.
