Description
BaybioManager is a desktop application designed for Parisgoshop.com, a cosmetics/organic products vendor. The company has launched several branch stores in online marketplace such as Ruten and Yahoo, each of them uses different admin panel. This application provides system admin the ability to manage transaction records and monitor inventory status in one place, without having to login to different online stores and manually check the quantities in stock. It also provide a simplfied workflow for the admin's daily operation, reducing the time required to manually insert transaction data into their ERP system.

From the technical point of view, the implementation of BaybioManager takes the future growth of the company into consideration and it is designed to be easy-to-maintain and easy-to-extend.

The GUI is built with a self-implemented MVC-like pattern, which separates the codes for manipulating data and displaying results. It requires extra classes to construct such design pattern, but will be easier for the developer to add new components in the future. Data Access Objects abstracts and isolates the underlying details of database connections from the business logic, and it will come in handy when the company needs to pull data from a newly launched online store or other datasource using a different type of database. High level concurrency api like SwingWorker also makes it simple to separate the EDT and worker threads.

*Currently the application is still under development*

Description
MySurvey is a semester long team project for the CS673 Software Engineering.  The goal is to create a web application that will provide user the ability to create, take and analyze online surveys.  Our target survey creators will be data analysts from corporations and from the education sector that are collecting data from their designated group of survey-takers.  The application will also provide pre-fabricated tools to speed up survey creation and publication.

There are many related work on the market, some are free , like SurveyMonkey which allow the user to login, create the survey, and view analytic reports, and some are costly but more sophisticated like Visimojo.  MySurvey offers a range of features geared towards both new survey creators and advanced users. This includes question templates to quickly get started in addition to customizable complex branching, time measurement for each question, and fine-tuned analytic reporting. Survey Monkey and Visimojo each offer many of the features weâ€™re planning in our application, but not all.

This project is implemented using the Yii framework. The development process follows a non-strict Agile development process, focused on features and collaboration. There are three iterations (excluding the initial planning phase), three weeks each. The initial phase focuses on planning, organization and defining requirements. The other three iterations are more typical Agile iterations.

Description
Wretch Downloader is a photo album downloader for www.wretch.cc (a blogging website owned by Yahoo, but it has been shut down since December 2013). The program allows user to automatically download open albums (publicly visible) of a specific account. It can be configured to manual or auto mode using different input parameters.

The program will parse the HTML information found on the given account, continuously search for the next available album, get the image url (the Mapping), and download the images automatically until it reaches the end of the last album.

Description
The Focus Area Detector is a program that combines image processing with Hadoop, capable of detecting and highlighting in-focus area of the input images in parallel. Built with the Hadoop MapReduce framework, it can be easily configured and deployed to a computer cluster or the cloud. The main task of this program involves calculating the difference of the RGB value between any two consecutive pixels, it is CPU intensive, but the computation can be broken into multiple small tasks and can be processed more efficiently using distributed system.

If an object is well-focused in the picture, the details on that object should be sharp and clear because the pixel's RGB value in that area changes rapidly. If the object is out of focus, that area seems blurred because the value changes gradually. For example, from black to dark gray, from dark gray to light gray, and from light gray to white.

The MapReduce task consists of three stages: First, the Builder writes the pixel information to a text-based file. Secondly, the Mapper reads the text input, calculates the difference, and writes the output to the next stage. Finally, the Reducer calculates the total score and a new image with hightlighted area will be created.

Description
The Interactive CPU Scheduling Simulator is an academic project originally designed and created by Jan Horjus and Matthew Powers-Freeling at BU. The application simulates the CPU scheduling mechanism in modern operating systems, it can be used as an instructional aid to analyze new algorithms or to help user better understand popular ones such as FIFO, SJF, Priority Scheduling, and Round-Robin.

In addition to the core functions in the first version, the interface will now print basic information of the process and dynamically calculate the waiting time and average turnaround time, making the simulator more informative and easier to understand. Three new algorithms are added to the simulator.
